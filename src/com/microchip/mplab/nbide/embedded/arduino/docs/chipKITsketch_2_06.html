Project Setup
The Import chipKIT Project dialog opens (Figure: Browse for Folder Containing Sketch) and displays the Project Setup options. Most of the information is filled in for you, though you can select different options, depending on your application. See Table: Project Setup Dialog.
Figure: Browse for Folder Containing Sketch
Table: Project Setup Dialog 
Setting
Description
Source Project Location
Click Browse to navigate to the folder containing the .ino file created by the Arduino IDE. Navigate to
C:\Program Files (x86)\Arduino\examples\01.Basics\Blink
Target Project Location
Click Browse to select the base directory where you would like the imported project stored. Typically, the actual directory will be a .X directory under this directory
Arduino Location
Where the importer found the installation of Arduino IDE
chipKIT Core Location
The location of the chipKIT Core files. The importer will use this directory to retrieve important files for building the core library file needed during building.
Target Board
Select the target board that the imported sketch will be run on. This should be a list of currently supported boards from the core.
Project Name
You can rename the Project name as desired, with the resulting Project Directory being renamed accordingly.
Project Directory
Location of your project directory.
Encoding
Select the character encoding that is appropriate for your use.
Copy all chipKIT dependencies
Checking the box below will create a stand-alone project that is logically separate from the chipKIT/Arduino environment. Leaving the box unchecked means that all source files and libraries will remain in the chipKIT/Arduino environment.
If the "Copy all chipKIT dependencies" option is selected, the importer will make a separate copy of the .ino file, converted to a .cpp file. After that, any changes made in either the .ino or the .cpp file will not be seen by the other. If the check box is cleared, then the project will maintain a link to the .ino file, and you will be able to edit in either IDE, with the changes reflected in the other.
Overwrite existing project
If you selected a project destination that already exists, this check box becomes active. Checking the box opens a dialog to confirm that you wish to overwrite it.
If the project is overwritten, any previous changes may be lost.
 
The plugin automatically assigns the specific PIC32 device based on the board that was selected.
